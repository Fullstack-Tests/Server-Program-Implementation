# 가이드 — 막힐 때 강사 SAMPLE 어디를 보면 되나

| 막히는 지점 | 참고 SAMPLE |
|:--|:--|
| Entity/@Id/@GeneratedValue, JpaRepository | 09_ORM_JPA |
| @Transactional (Spring Boot 기본 트랜잭션매니저) | 10_TX_JPA |
| @RestController / ResponseEntity / @CrossOrigin | 11_RESTCONTROLLER |
| @RequestBody @Valid + BindingResult 필드오류 처리 | 04_DATA_VALIDATION, 11_RESTCONTROLLER |
| @RestControllerAdvice / 커스텀 예외 @ResponseStatus | 05_EXCEPTION |
| DTO ↔ Entity 변환 toEntity()/from() | 07~11 공통 패턴 |

## 핵심 힌트
- `existsByName`, `existsByNameAndIdNot` 은 JPA 쿼리메서드라 **선언만** 하면 구현은 자동 생성된다.
- `registerBulk` 는 리스트를 for 로 돌며 저장하다가, 위반 데이터에서 `throw new MyBizException(...)` 하면
  `@Transactional` 이 **전체 롤백**한다. (저장 후 일부러 실패시켜 롤백을 테스트해 볼 것)
- `modify` 는 `findById` 로 영속 엔티티를 가져와 필드만 바꾸면 **변경감지(dirty checking)**로 update 된다.
- 목록은 JPA `Pageable`(page,size)로 조회하고 `.map(ProductDTO::from)` 으로 변환한다.
- 단위테스트는 `@SpringBootTest` + `@Transactional`(테스트 자동 롤백) 환경에서 작성한다.
