## 1. 업무 프로세스 개요
상품 관리 서비스의 주요 업무 흐름을 정리한다.

## 2. 세부 업무 프로세스 (요청 → 응답)
| 업무 | 경로 | 처리 흐름 |
|:--|:--|:--|
| 상품 등록 | POST /api/products | Controller(@Valid) → Service(중복검사·저장) → Repository → DB |
| 상품 목록 | GET /api/products |  |
| 상품 단건 | GET /api/products/{id} |  |
| 상품 수정 | PUT /api/products/{id} |  |
| 상품 삭제 | DELETE /api/products/{id} |  |
| 일괄 등록 | POST /api/products/bulk |  |
| 품절 점검(배치) | (스케줄러) |  |

## 3. 처리 흐름도 (텍스트)
```
[Client] → (HTTP) → [Controller] → [Service @Transactional] → [Repository(JPA)] → [MySQL]
                                          │
                                   [GlobalExceptionHandler]  ← 예외
```

## 4. 데이터 흐름 (상품 등록 예)
1. 클라이언트가 JSON(ProductDTO) 전송
2. @Valid 검증 (실패 시 400 + 필드 메시지)
3. Service: 상품명 중복 검사 → save
4. 응답: 200 + 등록 결과
