## ① 작업지시서(문제)

미완성 EX01(S-Product) 프로젝트의 핵심 클래스 TODO(ProductDTO, ProductRepository, ProductServiceImpl,
ProductRestController, ProductBatch, 테스트)를 강사 SAMPLE(09_ORM_JPA · 10_TX_JPA · 11_RESTCONTROLLER)
패턴으로 구현하여, 상품 REST CRUD(등록 @Valid 검증 → 저장 → 목록/단건/수정/삭제), 일괄등록(bulk:
한 트랜잭션·중간 실패 시 전체 롤백), 품절 점검 배치(@Scheduled)가 정상 동작하도록 완성하고
동작 화면(Postman)을 제출하시오. (DB 스키마 db/schema.sql + 일부 골격 클래스 제공)

## ② 내용(설명)

### [확인]
- 도메인   : 상품 (tbl_product: id, name, price, stock, category, create_at)
- 기능     : REST CRUD 5종(목록/단건/등록/수정/삭제) + 일괄등록(bulk) + 품절 점검 배치  ※ 검색 없음
- 흐름     : Controller → Service(인터페이스+Impl) → Repository (3계층 분리)
- 검증     : @Valid + BindingResult + FieldError (상품명/가격/재고/카테고리), 가격·재고 음수 차단
- 예외     : @RestControllerAdvice GlobalExceptionHandler + 커스텀 MyBizException(@ResponseStatus)
- 트랜잭션 : @Transactional (Spring Boot 자동설정 트랜잭션매니저 사용) — 등록/수정/삭제/일괄/배치
- 배치     : @EnableScheduling + @Scheduled (Spring 기본, 라이브러리 추가 없음) — 품절(stock=0) 점검
- DB 연동  : Spring Data JPA(JpaRepository) → MySQL testdb (파라미터 자동 바인딩)
- 환경     : JDK 21 / Spring Boot 3.5.x(REST API) / MySQL 8.x / 내장 Tomcat 8090
- 제공물   : db/schema.sql(CREATE+seed 3건), BatchConfig(@EnableScheduling), Product(Entity),
             ProductService(인터페이스), GlobalExceptionHandler, DemoApplication
             · ProductDTO 의 속성(필드)·검증 애너테이션은 제공(변환 메서드는 학생 구현)
- 학생 구현 :
   · ProductDTO         : toEntity() / from()
   · ProductRepository  : existsByName / existsByNameAndIdNot / countByStock (쿼리메서드 선언)
   · ProductServiceImpl : register / modify / remove / get / list / registerBulk (전 메서드)
   · ProductRestController : list / get / add / update / delete / addBulk (전 매핑)
   · ProductBatch       : checkSoldOut (배치 핵심 로직 — 상세 주석 참고)
   · 테스트             : ProductServiceTest / ProductBatchTest

### [핵심 구현 요구사항] — 강사 SAMPLE 09·10·11 패턴

1) ProductDTO (속성·검증 제공 / 변환 메서드 구현)
   - toEntity() : Product.builder() 로 필드 매핑하여 Entity 반환
   - from(Product) : ProductDTO.builder() 로 매핑하여 DTO 반환 (static)

2) ProductRepository (extends JpaRepository<Product, Long>)
   - boolean existsByName(String)               : 등록 중복 검사
   - boolean existsByNameAndIdNot(String, Long)  : 수정 시 자기 제외 중복 검사
   - long    countByStock(int)                   : 배치 품절(stock=0) 건수 조회
   - ※ 쿼리메서드는 선언만 하면 구현 자동 생성

3) ProductServiceImpl  (각 메서드에 @Transactional 적용)
   - register : 중복검사(existsByName) → createAt=now() → save → from 반환
   - modify   : findById(없으면 MyBizException) → 자기제외 중복검사 → 필드 수정(변경감지) update
   - remove   : existsById 확인 → deleteById
   - get      : @Transactional(readOnly=true) findById → from
   - list     : @Transactional(readOnly=true) findAll(pageable).map(ProductDTO::from)
   - registerBulk : List 순회 저장, 중복 시 MyBizException → 전체 롤백, 건수 반환

4) ProductRestController  (@RestController, /api/products)
   - list   : list(pageable) → content/page/totalPages/totalElements 를 Map 으로 200
   - get    : get(id) → 200
   - add    : @RequestBody @Valid + BindingResult, 오류 시 필드 메시지 400 / 통과 시 register → 200
   - update : @PathVariable id + 위와 동일 패턴 + modify
   - delete : remove(id) → 200
   - addBulk: @RequestBody List<ProductDTO> → registerBulk → count 반환

5) ProductBatch.checkSoldOut  (배치 — 진도 미학습, 파일 상세 주석 참고)
   - 메서드에 @Transactional(readOnly = true) 적용
   - productRepository.countByStock(0) 으로 품절 건수 반환
   - scheduleSoldOutCheck()(@Scheduled, 제공)가 60초마다 호출하여 로그 출력

6) 테스트
   - ProductServiceTest : register_정상(), registerBulk_중간실패_전체롤백()
   - ProductBatchTest   : checkSoldOut_동작()
